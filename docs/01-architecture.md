# 01. 아키텍처 개요

Quetta Agents MCP는 **3계층 시스템**으로 구성됩니다.

```
┌──────────────────────────────────────────────────────────────┐
│ Layer 1: Claude Code (사용자 인터페이스)                      │
│   - stdio 기반 MCP 프로토콜로 연결                            │
│   - quetta_*, quetta_auto 도구 호출                          │
└──────────────────────────────────────────────────────────────┘
                          │
                          ▼
┌──────────────────────────────────────────────────────────────┐
│ Layer 2: quetta-agents-mcp (클라이언트 라이브러리)            │
│   - 도구 디스패처                                              │
│   - 자연어 의도 분류 (quetta_auto)                            │
│   - 파이프라인 오케스트레이션                                  │
└──────────────────────────────────────────────────────────────┘
                          │
                          ▼
┌──────────────────────────────────────────────────────────────┐
│ Layer 3: Quetta 서버 (원격)                                   │
│   ┌──────────────┐  ┌──────────────┐  ┌──────────────────┐  │
│   │ LLM Gateway  │  │ Orchestrator │  │ RAG Service      │  │
│   │ (스마트 라우팅)│  │ (에이전트/태스크)│ │ (벡터 검색·저장) │  │
│   └──────────────┘  └──────────────┘  └──────────────────┘  │
│                                                              │
│   ┌──────────────┐  ┌──────────────────────────────────┐    │
│   │ File Storage │  │ Vector DB + Metadata DB           │    │
│   │ (TUS upload) │  │ (RAG chunks + 인제스트 메타)       │    │
│   └──────────────┘  └──────────────────────────────────┘    │
│                                                              │
│   LLM 백엔드: 로컬 Gemma4/DeepSeek-R1 + Claude + Gemini      │
│                                                              │
│   ┌──────────────────────────────────────────────────────┐  │
│   │ Remote Agents (사용자의 외부 PC)                      │  │
│   │ - WebSocket 역방향 연결 (포트포워딩 불필요)            │  │
│   │ - 화면 캡처 · 클릭 · 입력 · 셸                         │  │
│   │ - GPU 작업 (Nougat OCR, torch, CUDA 등)              │  │
│   └──────────────────────────────────────────────────────┘  │
└──────────────────────────────────────────────────────────────┘
```

## 컴포넌트별 역할

### Layer 1: Claude Code
- MCP 클라이언트. 사용자가 자연어로 요청.
- 도구를 직접 호출하거나 `quetta_auto` 한 번으로 전체 위임.

### Layer 2: quetta-agents-mcp
- 사용자의 PC에 설치되는 경량 클라이언트.
- **스마트 디스패처 `quetta_auto`**: 자연어 요청을 의도별로 분류해 적절한 도구로 자동 라우팅.
- **파이프라인 오케스트레이션**: 논문/설계도 분석 같은 복합 작업을 여러 서비스에 걸쳐 순차 조율.
- 단계별 실패에 대해 graceful degradation (부분 성공 결과 반환).

### Layer 3: Quetta 서버

| 서비스 | 역할 |
|--------|------|
| **LLM Gateway** | 질문 유형에 따라 최적 모델 자동 선택 (Gemma4 / DeepSeek-R1 / Claude) |
| **Orchestrator** | 전문 에이전트 관리, 태스크 실행, agent-skills 카탈로그 |
| **RAG Service** | 벡터 검색 + 인제스트 (논문/설계도/문서 통합 지식베이스) |
| **File Storage** | 대용량 파일 재개 가능 업로드 (TUS 프로토콜) |
| **Remote Agents** | 사용자의 외부 PC(GPU 포함)에 설치되는 에이전트 |

## 데이터 흐름 예시

### 예1: 논문 분석 + 이후 질의

```
사용자: "이 논문 분석해줘" + PDF 파일
  │
  ▼  MCP가 의도 자동 감지 → 논문 분석 파이프라인 실행
  │
  ├─ 1. PDF 업로드
  ├─ 2. GPU PC에서 Nougat OCR 실행 (수식·표 보존 Markdown)
  ├─ 3. Gemini Vision 으로 그림·도표 해석
  ├─ 4. Claude 가 전부 종합해 한글 리포트 생성
  └─ 5. 결과를 지식베이스(RAG)에 자동 저장

이후:
사용자: "이 논문의 핵심 수식 유도 과정?"
  │
  ▼  MCP가 "논문 질의" 의도로 감지
  │
  └─ 저장된 논문에서 관련 부분 검색 → Claude 답변 (출처 인용)
```

### 예2: 설계도 분석

```
사용자: "기계 조립도 GD&T 분석해줘" + PDF
  │
  ▼  MCP가 기계 설계 유형 자동 판별
  │
  ├─ 1. PDF 텍스트 추출 (주석·치수·BOM)
  ├─ 2. Gemini Vision 으로 GD&T 기호 · 조립 관계 해석
  ├─ 3. Claude 엔지니어링 리포트 생성
  └─ 4. 지식베이스 저장 → 이후 "M8 볼트 토크는?" 같은 질의 가능
```

## 성능 특성

| 작업 | 지연 | 비용 |
|------|------|------|
| `quetta_ask` (단순 질문) | 1–3초 | Gemma4 (무료, 로컬) |
| `quetta_ask` (복잡) | 5–15초 | Claude Sonnet |
| `quetta_multi_agent` | 30–90초 | Gemma4 × 3 + Claude 종합 |
| `quetta_remote_screenshot` | 1–2초 | 무료 |
| `quetta_gpu_exec` | 커맨드 의존 | 무료 (로컬 GPU) |
| `quetta_analyze_paper` (nougat 설치됨) | 3–10분 | Gemini + Claude |
| `quetta_analyze_paper` (nougat 첫 설치) | +10분 | — |
| `quetta_analyze_blueprint` | 30–90초 | Gemini + Claude |
| `quetta_paper_query` / `quetta_blueprint_query` | 3–8초 | Claude |

## 장애 내성 (Fault Tolerance)

- **GPU 에이전트 단절 시**: 자동 재시도 후 명확한 에러 메시지 반환
- **분석 단계 중 일부 실패 시**: 나머지 단계로 계속 진행하여 부분 결과 반환
- **저장 실패 시**: 분석 결과는 그대로 반환 (사용자가 참고 가능)
- **원격 에이전트 연결 끊김 시**: 자동 재연결 (네트워크 복구 후)

## 보안

- 모든 외부 API 호출은 Bearer 토큰 / API Key 로 인증
- 원격 에이전트 설치 링크: **24시간** 유효, 일회성
- 원격 에이전트 WebSocket: 토큰 인증 필수
- 모든 데이터 전송: HTTPS (TLS)
- 사용자 계정별 API 키로 권한 분리 (관리자가 발급)
